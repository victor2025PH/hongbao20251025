## 📝 注意事项

1. **环境变量**: 确保 `.env` 文件已配置，特别是：
   - `BOT_TOKEN` - Telegram Bot Token
   - `DATABASE_URL` - 数据库连接串
   - `ADMIN_WEB_USER` / `ADMIN_WEB_PASSWORD` - Web Admin 登录凭据

2. **数据库**: 首次运行需要初始化数据库：
   ```bash
   python init_db.py
   ```

3. **端口冲突**: 如果端口被占用，可以：
   - 修改 `docker-compose.yml` 中的端口映射
   - 或直接修改服务启动命令中的端口

4. **服务日志**: 查看服务日志以排查问题：
   - Web Admin: 查看控制台输出
   - MiniApp API: 查看控制台输出
   - Bot: 查看控制台输出

## 🔧 故障排查

### 服务无法启动

1. **检查 Python 环境**
   ```bash
   python --version
   pip list | grep -E "(fastapi|uvicorn|aiogram)"
   ```

2. **检查端口占用**
   ```powershell
   netstat -ano | Select-String -Pattern ":(8001|8080)"
   ```

3. **检查环境变量**
   ```bash
   # Windows PowerShell
   Get-Content .env
   ```

4. **查看错误日志**
   - 检查服务启动时的错误信息
   - 查看数据库连接是否正常
   - 检查依赖是否完整安装

### 健康检查失败

1. **等待服务完全启动**（通常需要 5-10 秒）
2. **检查服务是否在监听端口**
3. **检查防火墙设置**
4. **查看服务日志**
