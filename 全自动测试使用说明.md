# 🚀 全自动测试系统使用说明

## 📋 系统组件

已创建以下自动化工具：

1. **`auto_test.py`** - 全自动测试主程序
   - 自动检查环境
   - 自动安装依赖
   - 自动启动服务
   - 自动运行测试
   - 自动生成报告

2. **`auto_fix_common_errors.py`** - 自动修复常见错误
   - 修复导入错误
   - 修复数据库路径
   - 修复端口冲突
   - 修复环境变量

3. **`monitor_services.py`** - 实时监控服务
   - 监控日志
   - 检测错误
   - 自动修复
   - 健康检查

4. **`test_services.py`** - 服务健康检查
   - 快速测试服务状态
   - 显示服务端点

## 🎯 快速开始

### 方式 1: 一键启动（推荐）

```powershell
# 运行全自动测试（包含所有步骤）
python auto_test.py
```

这个命令会：
1. ✅ 检查 Python 版本
2. ✅ 安装所有依赖
3. ✅ 初始化数据库
4. ✅ 启动所有服务
5. ✅ 执行健康检查
6. ✅ 运行测试套件
7. ✅ 生成测试报告
8. ✅ 持续监控服务

### 方式 2: 分步执行

```powershell
# 步骤 1: 自动修复常见错误
python auto_fix_common_errors.py

# 步骤 2: 启动全自动测试
python auto_test.py

# 步骤 3: (可选) 在另一个终端启动监控
python monitor_services.py -d 600  # 监控 10 分钟
```

### 方式 3: 使用 PowerShell 脚本

```powershell
# 运行完整的自动化脚本
powershell -ExecutionPolicy Bypass -File run_full_auto_test.ps1
```

系统会自动修复以下问题：

1. **导入错误**

   - 自动安装缺失的包

3. **端口冲突**

4. **环境变量**
   - 从 .env.example 创建 .env
   - 创建基本配置

5. **日志配置**
   - 创建日志目录
   - 配置日志文件

## 📝 测试报告

测试完成后会生成 JSON 格式的报告：

- 文件名: `test_report_YYYYMMDD_HHMMSS.json`
- 包含内容:
  - 测试结果
  - 服务状态
  - 发现的错误
  - 应用的修复
  - 时间戳

## ⚠️ 注意事项

1. **首次运行**: 可能需要较长时间安装依赖
2. **端口占用**: 如果端口被占用，需要先停止占用进程
3. **环境变量**: 确保 .env 文件配置正确（特别是 BOT_TOKEN）
4. **数据库**: 首次运行会自动创建 SQLite 数据库
5. **服务停止**: 按 Ctrl+C 停止测试和监控

## 🐛 故障排查

### 问题 1: 服务启动失败

**解决方案**:
```powershell
# 1. 检查端口占用
netstat -ano | Select-String -Pattern ":(8001|8080)"

# 2. 运行自动修复
python auto_fix_common_errors.py

# 3. 查看详细错误
python -m uvicorn web_admin.main:app --host 0.0.0.0 --port 8001
```

### 问题 2: 依赖安装失败

**解决方案**:
```powershell
# 手动安装依赖
python -m pip install -r requirements.txt --upgrade

# 或单独安装缺失的包
python -m pip install fastapi uvicorn aiogram sqlalchemy
```

### 问题 3: 数据库错误

**解决方案**:
```powershell
# 重新初始化数据库
python -c "from models.db import init_db; init_db()"

# 或删除数据库文件重新创建
Remove-Item data.sqlite -ErrorAction SilentlyContinue
python init_db.py
```
