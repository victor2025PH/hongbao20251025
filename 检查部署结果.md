# ✅ 檢查部署結果

## 📊 修復腳本執行完成後的檢查

從終端輸出看到，修復腳本已經執行完成。現在請按照以下步驟檢查部署結果：

---

## 🔍 步驟 1: 檢查服務狀態

在服務器終端執行：

```bash
cd /opt/redpacket

# 查看所有服務狀態
docker compose -f docker-compose.production.yml ps
```

**預期結果：**
- 所有服務應該顯示 `Up` 或 `healthy` 狀態
- 沒有 `Exited` 或 `Restarting` 狀態

**如果服務未運行，請執行：**
```bash
# 查看詳細日誌
docker compose -f docker-compose.production.yml logs --tail 50
```

---

## 🧪 步驟 2: 測試健康檢查

```bash
# Web Admin 健康檢查
curl http://127.0.0.1:8000/healthz

# 預期輸出: {"ok": true, "ts": "..."}

# MiniApp API 健康檢查
curl http://127.0.0.1:8080/healthz

# 預期輸出: {"ok": true, "ts": "..."}

# 前端服務測試
curl http://127.0.0.1:3001 | head -20

# 預期輸出: HTML 內容
```

---

## 📋 步驟 3: 檢查服務日誌

```bash
cd /opt/redpacket

# 查看所有服務的日誌（最後30行）
docker compose -f docker-compose.production.yml logs --tail 30

# 查找錯誤
docker compose -f docker-compose.production.yml logs 2>&1 | grep -i error | tail -20
docker compose -f docker-compose.production.yml logs 2>&1 | grep -i fail | tail -20
```

## 🚨 如果仍然有問題

### 問題 1: 服務未啟動或狀態不正常

```bash
# 查看詳細日誌
docker compose -f docker-compose.production.yml logs

# 查看特定服務的日誌
docker compose -f docker-compose.production.yml logs db
docker compose -f docker-compose.production.yml logs web_admin
docker compose -f docker-compose.production.yml logs frontend

# 嘗試重啟服務
docker compose -f docker-compose.production.yml restart
```

### 問題 2: 健康檢查失敗

```bash
# 檢查服務是否正在運行
docker compose -f docker-compose.production.yml ps

# 檢查端口是否被占用
sudo netstat -tlnp | grep -E ':(8000|8080|3001|5432)'

# 檢查容器內部
docker compose -f docker-compose.production.yml exec web_admin curl http://localhost:8000/healthz
```

### 問題 3: 環境變量問題

```bash
# 檢查 .env.production 是否存在
ls -la .env.production

# 驗證環境變量
docker compose --env-file .env.production -f docker-compose.production.yml config | grep POSTGRES

# 如果環境變量仍然未設置，重新啟動時明確指定
docker compose --env-file .env.production -f docker-compose.production.yml down
docker compose --env-file .env.production -f docker-compose.production.yml up -d
```
