## 🔍 故障排查

### 问题 1: 服务无法启动

**可能原因**:
- 端口被占用
- 数据库连接失败
- 缺少依赖
- 环境变量未配置

**解决方案**:

```powershell
# 1. 检查端口占用
netstat -ano | Select-String -Pattern ":(8001|8080)"

# 2. 如果端口被占用，停止占用进程
# 找到进程 ID (PID)，然后：
Stop-Process -Id <PID> -Force

# 3. 检查数据库连接
python -c "from models.db import get_db; print('Database OK')"

# 4. 检查依赖
pip install -r requirements.txt

# 5. 查看详细错误信息
python -m uvicorn web_admin.main:app --host 0.0.0.0 --port 8001
# 查看控制台输出的错误信息
```

### 问题 2: 健康检查失败

**可能原因**:
- 服务未完全启动（等待时间不够）
- 服务启动失败但进程仍在运行
- 防火墙阻止

**解决方案**:

```powershell
# 1. 等待更长时间（10-15秒）
Start-Sleep -Seconds 15
python test_services.py

# 2. 检查服务进程
Get-Process | Where-Object {$_.ProcessName -like "*python*"}

# 3. 查看服务日志
# 如果使用后台启动，查看日志文件
Get-Content web_admin.log -Tail 50
Get-Content miniapp.log -Tail 50
```

### 问题 3: 数据库错误

**可能原因**:
- 数据库文件不存在
- 数据库权限问题
- 数据库连接串错误

**解决方案**:

```powershell
# 1. 检查数据库文件
if (Test-Path "data.sqlite") {
    Write-Host "数据库文件存在" -ForegroundColor Green
} else {
    Write-Host "数据库文件不存在，需要初始化" -ForegroundColor Yellow
    python init_db.py
}

# 2. 检查数据库连接
python -c "from models.db import engine; print(engine.url)"
```
