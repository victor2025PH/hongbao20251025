# 🔧 修復 Git 克隆問題

## 📊 問題診斷

從終端輸出看到：
- ✅ Git 克隆成功（625 個對象）
- ❌ 但只有 README.md 和 .git 目錄
- ❌ 缺少 `docker-compose.production.yml`
- ❌ 缺少 `deploy/` 目錄

**這說明克隆不完整！**

---

## 🔧 解決方案

### 方法 1: 清理並重新克隆（推薦）

在服務器上執行以下命令：

```bash
# 1. 進入項目目錄
cd /opt/redpacket

# 2. 備份 .env.production（如果已配置）
if [ -f .env.production ]; then
    cp .env.production /tmp/.env.production.bak
    echo "✅ 已備份 .env.production"
fi

# 3. 刪除當前目錄內容（保留 .env.production）
rm -rf .git README.md deploy frontend-next web_admin miniapp models services routers templates static

# 4. 重新克隆完整代碼
git clone https://github.com/victor2025PH/hongbao20251025.git .

# 5. 恢復 .env.production
if [ -f /tmp/.env.production.bak ]; then
    cp /tmp/.env.production.bak .env.production
    echo "✅ 已恢復 .env.production"
fi

# 6. 檢查關鍵文件
ls -la docker-compose.production.yml
ls -la deploy/
ls -la .env.production
```

### 方法 2: 檢查並拉取所有分支

```bash
cd /opt/redpacket

# 檢查遠程分支
git branch -r

# 檢查當前分支
git branch

# 切換到 master 分支（如果不在）
git checkout master

# 拉取所有文件
git fetch origin
git reset --hard origin/master

# 檢查文件
ls -la
```

### 方法 3: 檢查 .gitignore 是否忽略了文件

```bash
cd /opt/redpacket

# 檢查 .gitignore
cat .gitignore

# 強制添加所有文件
git add -f .
git status
```

## 🚀 修復後繼續部署

克隆修復後，執行：

```bash
cd /opt/redpacket

# 1. 修復 .env.production 配置
sed -i 's/NOWPAYMENTS_IPN_URLhttps/NOWPAYMENTS_IPN_URL=https/g' .env.production

# 2. 驗證文件存在
ls -la docker-compose.production.yml
ls -la deploy/scripts/deploy_on_server.sh

# 3. 開始部署
docker compose -f docker-compose.production.yml build
docker compose -f docker-compose.production.yml up -d

# 4. 等待並檢查
sleep 30
docker compose -f docker-compose.production.yml ps
```

## 💡 常見問題

### 問題 1: 克隆仍然不完整

**解決方案：**
```bash
# 檢查網絡連接
ping github.com

# 使用完整 URL 重新克隆
rm -rf /opt/redpacket
sudo mkdir -p /opt/redpacket
sudo chown $USER:$USER /opt/redpacket
cd /opt/redpacket
git clone --depth 1 https://github.com/victor2025PH/hongbao20251025.git .
```

### 問題 2: 權限問題

**解決方案：**
```bash
sudo chown -R $USER:$USER /opt/redpacket
chmod +x deploy/scripts/*.sh
```

### 問題 3: Git 倉庫損壞

**解決方案：**
```bash
cd /opt/redpacket
rm -rf .git
git init
git remote add origin https://github.com/victor2025PH/hongbao20251025.git
git fetch origin
git checkout -b master origin/master
```
