# 🔧 修復環境變量和網絡問題

## 📊 當前問題

從終端輸出看到：

1. **警告：環境變量未設置**
   ```
   WARN[0000] The "POSTGRES_USER" variable is not set. Defaulting to a blank string.
   WARN[0000] The "POSTGRES_DB" variable is not set. Defaulting to a blank string.
   ```

2. **錯誤：網絡超時**
   ```
   failed to copy: httpReadSeeker: failed open: failed to do request: 
   Get "https://production.cloudflare.docker.com/...": 
   net/http: TLS handshake timeout
   ```

---

## 🔧 解決方案

### 步驟 1: 檢查並修復 .env.production

```bash
cd /opt/redpacket

# 檢查 .env.production 是否存在
ls -la .env.production

# 檢查 POSTGRES_USER 和 POSTGRES_DB 是否設置
grep POSTGRES_USER .env.production
grep POSTGRES_DB .env.production

# 如果文件不存在，從備份恢復
if [ ! -f .env.production ]; then
    if [ -f /tmp/.env.production.bak ]; then
        cp /tmp/.env.production.bak .env.production
        sed -i 's/NOWPAYMENTS_IPN_URLhttps/NOWPAYMENTS_IPN_URL=https/g' .env.production
        echo "✅ .env.production 已恢復"
    else
        echo "❌ .env.production 不存在且無備份"
    fi
fi
```

### 步驟 2: 停止服務並清理

```bash
cd /opt/redpacket

# 停止所有服務
docker compose -f docker-compose.production.yml down

# 清理未完成的拉取
docker system prune -f
```

### 步驟 3: 重新拉取鏡像（處理網絡超時）

```bash
cd /opt/redpacket

# 設置 Docker 鏡像加速（如果需要）
# 或者直接重試拉取

# 重新拉取基礎鏡像
docker pull postgres:15-alpine
docker pull redis:7-alpine
docker pull prom/prometheus:latest
docker pull grafana/grafana:latest

# 如果拉取失敗，可以設置 Docker 鏡像加速器
```

### 步驟 4: 使用正確的環境變量文件啟動

```bash
cd /opt/redpacket

# 確保 .env.production 存在
[ -f .env.production ] || {
    echo "❌ .env.production 不存在，請先創建"
    exit 1
}

# 驗證環境變量
echo "檢查環境變量："
docker compose -f docker-compose.production.yml config | grep POSTGRES_USER
docker compose -f docker-compose.production.yml config | grep POSTGRES_DB

# 構建並啟動（使用 --env-file 明確指定）
docker compose --env-file .env.production -f docker-compose.production.yml up -d --build
```

## 🔍 如果網絡問題持續

### 設置 Docker 鏡像加速器（中國大陸）

```bash
# 創建或編輯 Docker daemon 配置
sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json <<-'EOF'
{
  "registry-mirrors": [
    "https://docker.mirrors.ustc.edu.cn",
    "https://hub-mirror.c.163.com"
  ]
}
EOF

# 重啟 Docker 服務
sudo systemctl daemon-reload
sudo systemctl restart docker

# 驗證配置
docker info | grep -A 10 "Registry Mirrors"
```

### 或者使用代理（如果有）

```bash
# 在 ~/.docker/config.json 中設置代理
mkdir -p ~/.docker
cat > ~/.docker/config.json <<'EOF'
{
  "proxies": {
    "default": {
      "httpProxy": "http://proxy.example.com:8080",
      "httpsProxy": "http://proxy.example.com:8080"
    }
  }
}
EOF
```
