- http://165.154.233.55:8000/healthz - 连接超时
- http://165.154.233.55:8080/healthz - 连接超时

---

## 🔍 可能原因

### 1. UCloud 安全组未配置（最可能）⭐

**问题**: 云服务商安全组未开放 8000 和 8080 端口

**验证方法**:
- 在浏览器中访问：http://165.154.233.55:8000/healthz
- 如果显示 `ERR_CONNECTION_TIMED_OUT`，很可能是安全组问题

**解决方案**: 见下方"配置 UCloud 安全组"章节

---

### 2. 服务未正常运行

**问题**: Docker 容器未启动或服务崩溃

**验证方法**:
- 需要 SSH 连接到服务器检查（如果 SSH 也连不上，说明安全组未开放 22 端口）

**解决方案**: 如果服务未运行，需要 SSH 连接后重启服务

---

### 3. 服务器防火墙未配置

**状态**: ✅ 已配置（之前已验证）
- 已开放 8000/tcp 端口
- 已开放 8080/tcp 端口

---

### 4. 端口绑定配置错误

**状态**: ✅ 已修复（之前已验证）
- Web Admin API: `0.0.0.0:8000:8000` ✅
- MiniApp API: `0.0.0.0:8080:8080` ✅

---

## 🔧 解决方案

### 方案 1: 配置 UCloud 安全组（最重要）⭐

#### 步骤 1: 登录 UCloud 控制台

1. 访问：**https://console.ucloud.cn/**
2. 使用您的账号登录

#### 步骤 2: 找到云主机

1. 在左侧菜单，点击 **"云主机 UHost"** 或 **"UHost"**
2. 在云主机列表中找到：**UHost-Sale**
   - **资源ID**: `uhost-lilim17se6os`
   - **公网IP**: `165.154.233.55`

#### 步骤 3: 进入安全组配置

1. 点击云主机名称 **"UHost-Sale"**，进入详情页
2. 在详情页中找到 **"安全组"** 或 **"防火墙"** 选项卡
3. 点击 **"配置规则"** 或 **"管理规则"** 按钮

#### 步骤 4: 添加入站规则

需要添加 **3 条规则**：

##### 规则 1: SSH (22 端口) - 用于远程管理

| 字段 | 值 |
|------|-----|
| 规则名称 | `SSH` |
| 方向 | `入站` 或 `Inbound` |
| 协议 | `TCP` |
| 端口范围 | `22` |
| 来源 | `0.0.0.0/0` 或您的IP |
| 动作 | `允许` 或 `Allow` |
| 优先级 | `默认` |
| 备注 | `SSH - 远程管理` |

##### 规则 2: Web Admin API (8000 端口)

| 字段 | 值 |
|------|-----|
| 规则名称 | `Web Admin API` |
| 方向 | `入站` 或 `Inbound` |
| 协议 | `TCP` |
| 端口范围 | `8000` |
| 来源 | `0.0.0.0/0` |
| 动作 | `允许` 或 `Allow` |
| 优先级 | `默认` |
| 备注 | `Web Admin API - 健康检查和API文档` |

##### 规则 3: MiniApp API (8080 端口)

| 字段 | 值 |
|------|-----|
| 规则名称 | `MiniApp API` |
| 方向 | `入站` 或 `Inbound` |
| 协议 | `TCP` |
| 端口范围 | `8080` |
| 来源 | `0.0.0.0/0` |
| 动作 | `允许` 或 `Allow` |
| 优先级 | `默认` |
| 备注 | `MiniApp API - 健康检查和API文档` |

#### 步骤 5: 保存并生效

1. 确认所有规则都已添加
2. 点击 **"保存"** 或 **"应用"** 按钮
3. **等待 1-2 分钟**让配置生效
4. 刷新页面，再次尝试访问

## 🔍 如果配置后仍然无法访问

### 检查 1: 确认规则已生效

1. 在 UCloud 控制台中查看安全组规则列表
2. 确认规则状态为"已生效"或"正常"
3. 确认规则已应用到正确的云主机

### 检查 2: 验证端口配置

1. 确认端口范围正确（8000 和 8080）
2. 确认协议是 TCP
3. 确认方向是"入站"（Inbound）

### 检查 3: 联系 UCloud 技术支持

如果配置正确但仍然无法访问：

1. 在 UCloud 控制台提交工单
2. 说明需要开放 8000 和 8080 端口
3. 提供云主机信息：UHost-Sale (uhost-lilim17se6os)

## 📝 重要提示

1. **安全组配置是最关键的一步** - 即使服务正常运行，如果安全组未开放端口，外部也无法访问

2. **配置生效需要时间** - 通常立即生效，但可能需要等待 1-2 分钟

3. **检查规则优先级** - 如果有拒绝规则，确保允许规则优先级更高

4. **建议只开放必要端口** - 如果需要更高安全性，可以将来源 IP 限制为特定 IP，而不是 `0.0.0.0/0`
