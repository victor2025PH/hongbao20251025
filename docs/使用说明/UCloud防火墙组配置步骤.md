# 🔐 UCloud 防火墙组配置步骤 - 解决"编辑规则"灰色问题

## 📋 问题描述

在"安全规则"选项卡中：
- ✅ 已进入"安全规则"选项卡
- ✅ 看到了"外网防火墙"部分
- ❌ "编辑规则"按钮是灰色的，无法点击
- ❌ 没有"添加规则"或"新建规则"按钮

---

## 🔍 问题原因

"编辑规则"按钮灰色是因为：
- 当前使用的是**系统默认防火墙**或**预定义的防火墙组**
- 默认防火墙通常**不允许直接编辑规则**
- 需要先**创建或更换为自定义防火墙组**

---

## 🔧 解决方案：创建新的防火墙组

### 步骤 1: 点击"更换外网防火墙"按钮

在"外网防火墙"部分，找到蓝色的 **"更换外网防火墙"** 按钮，点击它。

```
[更换外网防火墙] [编辑规则(灰色)]
 ↑
点击这里
```

---

### 步骤 2: 创建新的防火墙组

点击"更换外网防火墙"后，会弹出对话框，您会看到：

#### 选项 A: 创建新的防火墙组（推荐）

1. 点击 **"新建防火墙组"** 或 **"创建防火墙组"** 按钮
2. 填写防火墙组信息：
   ```
   防火墙组名称: redpacket-api-firewall
   备注: Web Admin API 和 MiniApp API 防火墙
   ```
3. 点击 **"确定"** 或 **"创建"**

#### 选项 B: 使用现有防火墙组

如果有现有的自定义防火墙组，也可以选择它。

---

### 步骤 3: 配置防火墙规则

创建新的防火墙组后，系统可能会：
- 自动跳转到防火墙组配置页面
- 或者在对话框中直接显示规则配置选项

**此时，您应该可以看到"添加规则"或"新建规则"按钮了！**

---

### 步骤 4: 添加入站规则

在防火墙组配置页面中，点击 **"添加规则"** 或 **"新建规则"** 按钮。

#### 规则 1: Web Admin API (8000 端口)

点击"添加规则"后，填写以下信息：

```
规则名称: Web Admin API
方向: 入站（或 Inbound）
协议: TCP
端口范围: 8000（或 8000-8000）
来源: 0.0.0.0/0
动作: 接受（或 Allow）
优先级: 高（或 High）
备注: Web Admin API - 健康检查和API文档
```

点击 **"确定"** 或 **"保存"**

#### 规则 2: MiniApp API (8080 端口)

再次点击 **"添加规则"** 按钮，填写以下信息：

```
规则名称: MiniApp API
方向: 入站（或 Inbound）
协议: TCP
端口范围: 8080（或 8080-8080）
来源: 0.0.0.0/0
动作: 接受（或 Allow）
优先级: 高（或 High）
备注: MiniApp API - 健康检查和API文档
```

点击 **"确定"** 或 **"保存"**

---

### 步骤 5: 保存并应用到云主机

1. 确认两条规则都已添加到列表中
2. 点击 **"确定"** 或 **"保存"** 按钮
3. 如果系统提示，选择 **"应用到云主机"** 或 **"绑定到云主机"**
4. 选择 **UHost-Sale** 云主机
5. 点击 **"确定"** 或 **"应用"**

---

### 步骤 6: 等待生效

1. 规则配置完成后，**等待 1-2 分钟**让配置生效
2. 刷新浏览器页面
3. 返回云主机详情页，检查"安全规则"选项卡
4. 应该可以看到新添加的规则了

## 📝 替代方案：通过"网络"选项卡配置

如果"更换外网防火墙"后仍然无法添加规则，可以尝试：

### 方法 1: 通过"网络"选项卡

1. 在云主机详情页，点击 **"网络"** 选项卡
2. 找到 **"防火墙组"** 或 **"安全组"** 部分
3. 点击 **"管理防火墙组"** 或 **"配置防火墙"**
4. 在防火墙组管理页面中创建新规则

### 方法 2: 通过左侧导航栏

1. 在左侧导航栏中，查找 **"防火墙"** 或 **"安全组"** 相关菜单
2. 点击进入防火墙组管理页面
3. 创建新的防火墙组并配置规则
4. 然后返回云主机，应用该防火墙组

---

## ⚠️ 注意事项

1. **保留现有规则**: 创建新防火墙组时，建议保留现有的SSH(22)、HTTP(80)、HTTPS(443)等规则
2. **优先级设置**: 如果有多个规则，确保允许规则优先级高于拒绝规则
3. **生效时间**: 规则配置后需要等待 1-2 分钟才会生效
