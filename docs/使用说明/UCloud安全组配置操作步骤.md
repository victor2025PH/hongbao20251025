### 步骤 4: 进入安全组配置

点击"安全组"选项卡后，您会看到：

点击 **"配置规则"** 或 **"管理规则"** 按钮。

---

### 步骤 5: 添加入站规则

在安全组规则配置页面中：

1. **找到"入站规则"或"Inbound Rules"部分**
2. **点击"添加规则"或"Add Rule"按钮**
3. **填写第一条规则（Web Admin API）**：

   ```
   规则名称: Web Admin API
   方向: 入站（或 Inbound）
   协议: TCP
   端口范围: 8000（或 8000-8000）
   来源: 0.0.0.0/0
   动作: 允许（或 Allow）
   优先级: 默认
   备注: Web Admin API
   ```

4. **点击"确定"或"保存"**

5. **再次点击"添加规则"按钮**

6. **填写第二条规则（MiniApp API）**：

   ```
   规则名称: MiniApp API
   方向: 入站（或 Inbound）
   协议: TCP
   端口范围: 8080（或 8080-8080）
   来源: 0.0.0.0/0
   动作: 允许（或 Allow）
   优先级: 默认
   备注: MiniApp API
   ```

7. **点击"确定"或"保存"**

---

### 步骤 6: 保存并生效

1. 确认两条规则都已添加到列表中
2. 点击页面上的 **"保存"** 或 **"应用"** 按钮
3. **等待 1-2 分钟**让配置生效
4. 配置完成后，刷新浏览器，重新访问 API

## ⚠️ 常见问题

### Q1: 找不到"安全组"选项卡？

**解决方法：**
- 确保您点击的是"详情"按钮，而不是"登录"或其他按钮
- 在详情页顶部查找选项卡
- 如果看不到，可能需要在详情页的"网络"或"安全"部分查找

### Q2: 找不到"添加规则"按钮？

**解决方法：**
- 在安全组配置页面查找"入站规则"或"Inbound Rules"部分
- 通常按钮会显示为"添加规则"、"新建规则"、"Add Rule"等
- 有些界面可能在规则列表顶部或右上角

### Q3: 端口配置格式不对？

**解决方法：**
- 如果系统要求范围，填写：`8000-8000` 或 `8080-8080`
- 如果系统只允许单个端口，填写：`8000` 或 `8080`
- 来源IP填写：`0.0.0.0/0`（允许所有IP）

### Q4: 保存后仍然无法访问？

**解决方法：**
- 等待 1-2 分钟让配置生效
- 刷新浏览器页面
- 检查规则是否已添加到列表中
- 确认规则状态为"已生效"或"正常"
