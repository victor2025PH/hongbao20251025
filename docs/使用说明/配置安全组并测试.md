# 🔐 配置安全组并测试 - 完整指南

## 📋 概述

由于云服务商安全组需要在控制台配置，本指南提供详细的配置步骤和测试方法。

---

## 🔧 UCloud 安全组配置步骤

### 步骤 1: 登录 UCloud 控制台

1. 访问：**https://console.ucloud.cn/**
2. 使用您的账号登录

---

### 步骤 2: 找到云主机

1. 在左侧菜单，点击 **"云主机 UHost"** 或 **"UHost"**
2. 在云主机列表中找到：**UHost-Sale**
   - **资源ID**: `uhost-lilim17se6os`
   - **公网IP**: `165.154.233.55`
   - **状态**: 运行中

---

### 步骤 3: 进入安全组配置

1. 点击云主机名称 **"UHost-Sale"**，进入详情页
2. 在详情页中找到 **"安全组"** 或 **"防火墙"** 选项卡
3. 点击 **"配置规则"** 或 **"管理规则"** 按钮

---

### 步骤 4: 添加入站规则

#### 规则 1: Web Admin API (8000 端口)

点击 **"添加规则"** 或 **"新建规则"**，填写以下信息：

| 字段 | 值 |
|------|-----|
| 规则名称 | `Web Admin API` |
| 方向 | `入站` 或 `Inbound` |
| 协议 | `TCP` |
| 端口范围 | `8000` 或 `8000-8000` |
| 来源 | `0.0.0.0/0` (允许所有IP) |
| 动作 | `允许` 或 `Allow` |
| 优先级 | `默认` |
| 备注 | `Web Admin API - 健康检查和API文档` |

点击 **"确定"** 或 **"保存"**

#### 规则 2: MiniApp API (8080 端口)

再次点击 **"添加规则"**，填写以下信息：

| 字段 | 值 |
|------|-----|
| 规则名称 | `MiniApp API` |
| 方向 | `入站` 或 `Inbound` |
| 协议 | `TCP` |
| 端口范围 | `8080` 或 `8080-8080` |
| 来源 | `0.0.0.0/0` (允许所有IP) |
| 动作 | `允许` 或 `Allow` |
| 优先级 | `默认` |
| 备注 | `MiniApp API - 健康检查和API文档` |

点击 **"确定"** 或 **"保存"**

---

### 步骤 5: 应用规则

1. 确认两条规则都已添加成功
2. 点击 **"应用"** 或 **"生效"** 按钮（如果有）
3. **等待 1-2 分钟**让配置完全生效

## 🔍 故障排除

### 问题 1: 连接超时 (ERR_CONNECTION_TIMED_OUT)

**可能原因：**
1. 安全组规则未配置或未生效
2. 服务器防火墙未开放端口（已配置 ✅）
3. 服务未正常运行（已确认 ✅）

**解决方法：**
1. 确认安全组规则已添加并保存
2. 等待 1-2 分钟让配置生效
3. 检查规则是否应用到正确的云主机
4. 重新运行测试脚本

---

### 问题 2: 连接被拒绝 (ERR_CONNECTION_REFUSED)

**可能原因：**
- 服务未正常运行
- 端口配置错误

**解决方法：**
- 检查服务状态（需要SSH访问服务器）
- 验证端口配置

---

### 问题 3: SSH 连接超时

如果 SSH (22 端口) 也连接超时，可能：
1. 安全组未开放 22 端口
2. 服务器有问题

**解决方法：**
- 检查安全组是否有 22 端口规则
- 如果 SSH 无法连接，可能需要通过控制台的 VNC 访问
