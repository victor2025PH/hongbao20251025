# 🔐 UCloud 安全组配置详细步骤

## 📋 快速配置步骤

### 步骤 1: 登录 UCloud 控制台

1. 访问：https://console.ucloud.cn/
2. 使用您的账号登录

---

### 步骤 2: 找到云主机

1. 在控制台左侧菜单，找到 **"云主机 UHost"** 或 **"UHost"**
2. 点击进入云主机列表
3. 找到云主机：**UHost-Sale**
   - 资源ID: `uhost-lilim17se6os`
   - 公网IP: `165.154.233.55`

---

### 步骤 3: 进入安全组配置

1. 点击云主机名称 **"UHost-Sale"**，进入详情页
2. 在详情页中找到 **"安全组"** 或 **"防火墙"** 选项卡
3. 点击 **"配置规则"** 或 **"管理规则"** 按钮

---

### 步骤 4: 添加入站规则

#### 规则 1: Web Admin API (8000 端口)

点击 **"添加规则"** 或 **"新建规则"**，填写：

- **规则名称**: `Web Admin API`
- **方向**: `入站` 或 `Inbound`
- **协议**: `TCP`
- **端口范围**: `8000` 或 `8000-8000`
- **来源**: `0.0.0.0/0` (允许所有IP) 或特定IP
- **动作**: `允许` 或 `Allow`
- **优先级**: `默认` 或 `100`
- **备注**: `Web Admin API - 健康检查和API文档`

点击 **"确定"** 或 **"保存"**

#### 规则 2: MiniApp API (8080 端口)

再次点击 **"添加规则"**，填写：

- **规则名称**: `MiniApp API`
- **方向**: `入站` 或 `Inbound`
- **协议**: `TCP`
- **端口范围**: `8080` 或 `8080-8080`
- **来源**: `0.0.0.0/0` (允许所有IP) 或特定IP
- **动作**: `允许` 或 `Allow`
- **优先级**: `默认` 或 `100`
- **备注**: `MiniApp API - 健康检查和API文档`

点击 **"确定"** 或 **"保存"**

---

### 步骤 5: 应用规则

1. 确认规则已添加成功
2. 规则通常立即生效（如果没有，点击"应用"或"生效"按钮）
3. 等待 1-2 分钟让配置完全生效

---

### 步骤 6: 验证配置

配置完成后，使用以下方法测试：

#### 浏览器测试

直接访问：
- Web Admin API: http://165.154.233.55:8000/healthz

## ⚠️ 注意事项

1. **安全性**: 如果只需要特定IP访问，请将来源改为您的IP，而不是 `0.0.0.0/0`
2. **规则顺序**: 如果有拒绝规则，确保允许规则优先级更高
3. **生效时间**: 规则通常立即生效，但可能需要等待 1-2 分钟
4. **端口冲突**: 确保没有其他规则阻止这些端口

## 🔍 故障排除

### 问题 1: 仍然无法访问

**检查清单：**
- [ ] 规则是否已正确添加并保存
- [ ] 规则是否已应用到正确的云主机
- [ ] 是否等待了 1-2 分钟让配置生效
- [ ] 服务器防火墙是否已开放端口（已确认 ✅）
- [ ] 服务是否正常运行（已确认 ✅）

### 问题 2: 规则添加失败

- 检查是否有权限添加规则
- 检查规则数量是否达到上限
- 尝试删除旧规则后重新添加

### 问题 3: 访问被拒绝（非超时）

- 检查规则方向是否为"入站"
- 检查动作是否为"允许"
- 检查来源IP是否包含您的IP

---

3. **检查网络连接**
   - 确保您的网络可以访问该IP
   - 尝试 ping 165.154.233.55
