# 🔥 防火墙和安全组配置指南

## ❌ 问题：连接超时 (ERR_CONNECTION_TIMED_OUT)

访问 `http://165.154.233.55:8000/healthz` 或 `http://165.154.233.55:8080/healthz` 时出现连接超时错误。

---

## 🔍 可能的原因

1. **服务器防火墙未开放端口**
2. **云服务商安全组规则未配置**
3. **服务未正常运行**（已排除，服务正常运行）

---

## ✅ 解决方案

### 方案 1: 配置服务器防火墙

#### Ubuntu/Debian (ufw)

```bash
# 登录服务器
ssh ubuntu@165.154.233.55

# 开放端口
sudo ufw allow 8000/tcp
sudo ufw allow 8080/tcp

# 确保防火墙已启用
sudo ufw enable

# 查看状态
sudo ufw status
```

#### CentOS/RHEL (firewalld)

```bash
# 开放端口
sudo firewall-cmd --permanent --add-port=8000/tcp
sudo firewall-cmd --permanent --add-port=8080/tcp

# 重新加载防火墙
sudo firewall-cmd --reload

# 查看状态
sudo firewall-cmd --list-ports
```

---

### 方案 2: 配置云服务商安全组（重要！）

#### UCloud 云服务商配置步骤

1. **登录 UCloud 控制台**
   - 访问：https://console.ucloud.cn/
   - 登录您的账号

2. **找到云主机**
   - 进入"云主机 UHost"页面
   - 找到云主机：**UHost-Sale** (资源ID: `uhost-lilim17se6os`)

3. **配置安全组规则**
   - 点击云主机，进入详情页
   - 找到"安全组"或"防火墙"选项
   - 点击"配置规则"或"添加规则"

4. **添加入站规则**

   **规则 1: Web Admin API (8000 端口)**
   - 方向：**入站**
   - 协议：**TCP**
   - 端口：**8000**
   - 来源：**0.0.0.0/0**（允许所有IP）或特定IP
   - 动作：**允许**
   - 备注：Web Admin API

   **规则 2: MiniApp API (8080 端口)**
   - 方向：**入站**
   - 协议：**TCP**
   - 端口：**8080**
   - 来源：**0.0.0.0/0**（允许所有IP）或特定IP
   - 动作：**允许**
   - 备注：MiniApp API

5. **保存规则**
   - 点击"确定"或"保存"
   - 规则通常立即生效

---

#### 其他云服务商

**阿里云 ECS**
- 进入"云服务器 ECS" → "安全组"
- 找到对应的安全组，点击"配置规则"
- 添加"入方向"规则

**腾讯云 CVM**
- 进入"云服务器" → "安全组"
- 找到对应的安全组，点击"修改规则"
- 添加"入站规则"

**AWS EC2**
- 进入 EC2 控制台 → "Security Groups"
- 选择对应的安全组 → "Inbound rules"
- 点击"Edit inbound rules"添加规则

## 🧪 验证配置

### 1. 检查服务器防火墙

```bash
# Ubuntu/Debian
sudo ufw status | grep -E "8000|8080"

# CentOS/RHEL
sudo firewall-cmd --list-ports | grep -E "8000|8080"
```

### 2. 检查端口监听

```bash
# 在服务器上执行
netstat -tlnp | grep -E ":8000|:8080"

# 应该看到：
# 0.0.0.0:8000   (Web Admin)
# 0.0.0.0:8080   (MiniApp API)
```

### 3. 测试本地访问

1. **检查云服务商安全组规则**（最可能的原因）

## 📝 临时解决方案（仅用于测试）

如果需要快速测试，可以临时关闭防火墙（不推荐生产环境）：

```bash
# Ubuntu/Debian
sudo ufw disable

# CentOS/RHEL
sudo systemctl stop firewalld
```

**⚠️ 注意**：测试完成后请重新启用防火墙并正确配置规则。
