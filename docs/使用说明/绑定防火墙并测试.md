## ✅ 当前状态

防火墙规则已成功导入：
- ✅ 规则数量：2
- ✅ TCP:8000 - Web Admin API
- ✅ TCP:8080 - MiniApp API
- ⚠️  绑定资源数量：0（需要绑定到云主机）

## 🎯 下一步：绑定防火墙到云主机（最重要！）

### ⚠️ 重要提示

即使规则配置正确，如果防火墙未绑定到云主机，外部仍然无法访问！

---

### 步骤 1: 返回云主机列表

1. 在左侧导航栏，点击 **"云主机 UHost"**
2. 找到 **UHost-Sale** 云主机

---

### 步骤 2: 进入云主机详情页

1. 点击 **UHost-Sale** 这一行的 **"详情"** 按钮
2. 进入详情页后，点击 **"安全规则"** 选项卡

---

### 步骤 3: 更换防火墙

1. 在"安全规则"选项卡中，找到 **"更换外网防火墙"** 按钮
2. 点击该按钮

---

### 步骤 4: 选择防火墙

1. 在弹出的对话框中，选择 **`redpacket-api-firewall`**
2. 点击 **"确定"** 或 **"应用"**

---

### 步骤 5: 验证绑定

绑定完成后：

1. **方法 1: 在云主机详情页验证**
   - 在"安全规则"选项卡的"资源绑定"部分
   - 应该可以看到绑定的防火墙信息

2. **方法 2: 返回防火墙列表验证**
   - 返回"外网防火墙"列表页面
   - 找到 `redpacket-api-firewall`
   - 确认"是否绑定资源"列显示为 **"是"**
   - 确认"绑定资源数量"显示为 **"1"**

---

## ⏰ 步骤 6: 等待生效

绑定完成后：
- **等待 1-2 分钟**让配置生效
- 配置生效后，防火墙规则才会应用到云主机

---

配置生效后，使用以下方法测试：

### 方法 1: 使用测试脚本（推荐）

在 PowerShell 中运行：
```powershell
.\test-api.ps1
```

### 方法 2: 浏览器测试

直接访问：
- **Web Admin API**: http://165.154.233.55:8000/healthz
- **MiniApp API**: http://165.154.233.55:8080/healthz

**预期响应**：

```powershell

# MiniApp API

```

## 🔍 如果测试仍然失败

### 检查 1: 确认绑定状态

如果测试仍然失败，首先检查：
- 防火墙是否已绑定到云主机（"是否绑定资源"应该显示为"是"）
- "绑定资源数量"应该显示为"1"

### 检查 2: 检查生效时间

- 配置后需要等待 1-2 分钟才会生效
- 如果刚绑定完，请等待后再测试

### 检查 3: 检查服务状态

如果绑定正确但仍然无法访问，可能是服务未运行：
- 需要 SSH 连接到服务器检查服务状态
- 检查 Docker 容器是否正常运行
