## 二、目标目录结构设计方案

### 2.1 设计原则

1. **保持运行逻辑不变**：
   - `docker-compose.yml`, `Dockerfile*` 保持在根目录（Docker 约定）
   - 核心代码目录（`core/`, `services/`, `routers/`, `models/`, `web_admin/`, `miniapp/`, `frontend-next/`）保持原位
   - `docs/deployment/` 下的脚本路径保持不变（已被 CI/CD 和 README 引用）

2. **分类清晰**：
   - 部署/运维脚本 → `infra/` 或 `ops/`
   - 开发/测试工具脚本 → `scripts/`
   - 历史/一次性脚本 → `legacy/`
   - 数据文件 → `data/`
   - 文档已分类 → `docs/`（保持现有分类）

3. **便于维护**：
   - 新脚本放在对应分类目录
   - 历史脚本归档到 `legacy/`，但保留文件名便于追溯
