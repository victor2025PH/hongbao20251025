# 🚀 下一步部署操作

根據當前狀態，您需要執行以下操作：

✅ Docker 已安裝並測試成功

## 🔧 步驟 1: 修復問題

### 問題 1: 修復 .env.production 配置錯誤

在 `.env.production` 文件中，有一行缺少等號：

**錯誤：**
```
NOWPAYMENTS_IPN_URLhttps://cd70-180-190-191-168.ngrok-free.app/nowpayments-ipn
```

**正確：**
```
NOWPAYMENTS_IPN_URL=https://cd70-180-190-191-168.ngrok-free.app/nowpayments-ipn
```

**修復方法：**

在服務器上執行：

```bash
cd /opt/redpacket
sed -i 's/NOWPAYMENTS_IPN_URLhttps/NOWPAYMENTS_IPN_URL=https/g' .env.production

# 驗證修復
grep NOWPAYMENTS_IPN_URL .env.production
```

或者使用 nano 編輯器手動修復：

```bash
nano .env.production
# 找到 NOWPAYMENTS_IPN_URLhttps 這行
# 改為 NOWPAYMENTS_IPN_URL=https://...
# Ctrl+O 保存，Ctrl+X 退出
```

## 🚀 步驟 2: 開始部署

### 方法 1: 使用 Docker Compose 直接部署（推薦）

```bash
cd /opt/redpacket

# 1. 構建 Docker 鏡像（首次部署需要）
docker compose -f docker-compose.production.yml build

# 2. 啟動所有服務
docker compose -f docker-compose.production.yml up -d

# 3. 等待服務啟動（30-60秒）
sleep 30

# 4. 查看服務狀態
docker compose -f docker-compose.production.yml ps
```

### 方法 2: 使用部署腳本（如果可用）

```bash
cd /opt/redpacket

# 檢查腳本是否存在
ls -la deploy/scripts/deploy_on_server.sh

# 如果存在，執行
chmod +x deploy/scripts/deploy_on_server.sh
bash deploy/scripts/deploy_on_server.sh
```

---

## ✅ 步驟 3: 驗證部署

### 1. 檢查所有服務狀態

```bash
docker compose -f docker-compose.production.yml ps
```

**預期輸出：** 所有服務應該顯示 `Up` 或 `healthy` 狀態

### 2. 測試健康檢查

```bash
# 數據庫健康檢查
docker compose -f docker-compose.production.yml exec db pg_isready -U redpacket -d redpacket

# Web Admin 健康檢查
curl http://127.0.0.1:8000/healthz
# 應該返回: {"ok": true, "ts": "..."}

# MiniApp API 健康檢查
curl http://127.0.0.1:8080/healthz
# 應該返回: {"ok": true, "ts": "..."}

# 前端服務測試
curl http://127.0.0.1:3001 | head -20
```

### 3. 查看服務日誌

```bash
# 查看所有服務日誌
docker compose -f docker-compose.production.yml logs -f

# 查看特定服務日誌
docker compose -f docker-compose.production.yml logs -f web_admin
docker compose -f docker-compose.production.yml logs -f frontend
```

## 🔍 故障排查

### 如果服務啟動失敗

```bash
# 查看詳細日誌
docker compose -f docker-compose.production.yml logs --tail 100

# 檢查配置
docker compose -f docker-compose.production.yml config

# 檢查環境變量
cat .env.production | grep -v "^#" | grep "="
```

### 如果端口被占用

```bash
# 檢查端口占用
sudo netstat -tlnp | grep -E ':(8000|8080|3001|5432)'

# 停止占用端口的服務
sudo systemctl stop <service-name>
```

### 如果數據庫連接失敗

```bash
# 檢查數據庫容器
docker compose -f docker-compose.production.yml ps db

# 查看數據庫日誌
docker compose -f docker-compose.production.yml logs db

# 測試數據庫連接
docker compose -f docker-compose.production.yml exec db psql -U redpacket -d redpacket -c "SELECT 1;"
```
