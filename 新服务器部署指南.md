## 🐳 步骤 2: 安装 Docker 和 Docker Compose

### ⚡ 方法 1: 一键执行所有命令（推荐，最简单）✨

**可以直接复制以下所有命令，一次性粘贴到服务器终端执行：**

```bash
# 一键安装 Docker 和 Docker Compose（复制所有命令，一次性粘贴执行）
sudo apt-get update && \
sudo apt-get install -y curl wget git && \
curl -fsSL https://get.docker.com -o /tmp/get-docker.sh && \
sudo sh /tmp/get-docker.sh && \
sudo usermod -aG docker $USER && \
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose && \
sudo chmod +x /usr/local/bin/docker-compose && \
sudo systemctl start docker && \
sudo systemctl enable docker && \
docker --version && \
docker-compose --version && \
echo "✅ Docker 安装完成！" && \
echo "⚠️  注意: 请重新登录 SSH 或执行 'newgrp docker' 以使权限生效"
```

**📋 使用方法：**
1. **复制上面的所有命令**（从 `sudo apt-get update` 开始到最后的 `echo` 结束，包括所有行）
2. **在服务器终端中右键粘贴**（或按 `Shift+Insert`）
3. **按 Enter 执行**
4. 命令会自动依次执行，无需多次复制

**⚠️ 重要提示：**
- 如果提示输入密码，输入您的 sudo 密码
- 安装完成后，建议重新登录 SSH 以使 docker 组权限生效
- 或者执行：`newgrp docker`

## ⚙️ 步骤 4: 配置环境变量

### 在服务器上执行：

```bash
# 1. 进入项目目录
cd /opt/redpacket

# 2. 创建环境变量文件（基于模板）
cat > .env.production << 'EOF'
# PostgreSQL 数据库配置（必需）
POSTGRES_USER=redpacket
POSTGRES_PASSWORD=你的强密码  # ⚠️ 请修改为强密码！
POSTGRES_DB=redpacket

# 数据库连接 URL（可选，会自动生成）
# DATABASE_URL=postgresql+psycopg2://redpacket:你的密码@db:5432/redpacket

# Telegram Bot 配置（必需）
BOT_TOKEN=你的BotToken
ADMIN_IDS=你的Telegram用户ID
SUPER_ADMINS=你的Telegram用户ID

# 其他配置
FLAG_ENABLE_PUBLIC_GROUPS=1
DEBUG=false
TZ=Asia/Manila

# Web Admin 配置
ADMIN_WEB_USER=admin
ADMIN_WEB_PASSWORD=你的管理员密码
ADMIN_SESSION_SECRET=change_me_at_least_32_chars_random_string

# MiniApp 配置
MINIAPP_JWT_SECRET=change_me_to_secure_value
MINIAPP_JWT_ISSUER=miniapp
MINIAPP_JWT_EXPIRE_SECONDS=7200

# NOWPayments 配置（如果使用）
NOWPAYMENTS_API_KEY=你的API密钥
NOWPAYMENTS_IPN_SECRET=你的IPN密钥
NOWPAYMENTS_IPN_URL=https://你的域名/api/v1/ipn/nowpayments
EOF

# 3. 编辑配置文件（修改密码和其他配置）
nano .env.production
# 或使用 vim: vim .env.production
```

**⚠️ 重要：必须修改以下配置：**

1. `POSTGRES_PASSWORD` - 数据库密码（强密码，建议 16 位以上）
2. `BOT_TOKEN` - Telegram Bot Token
3. `ADMIN_IDS` - 您的 Telegram 用户 ID
4. `ADMIN_WEB_PASSWORD` - Web 管理员密码
5. `ADMIN_SESSION_SECRET` - 随机字符串（至少 32 个字符）
6. `MINIAPP_JWT_SECRET` - JWT 密钥（随机字符串）

**生成随机密钥的方法：**

```bash
# 生成 32 位随机字符串
openssl rand -hex 32
```

## 🚀 步骤 5: 部署服务

### 方法 1: 使用自动部署脚本（推荐）

```bash
# 在项目目录中执行
cd /opt/redpacket

# 给脚本添加执行权限
chmod +x deploy/scripts/deploy_on_server.sh

# 执行部署脚本
bash deploy/scripts/deploy_on_server.sh
```

### 方法 2: 手动执行部署命令

```bash
# 1. 进入项目目录
cd /opt/redpacket

# 2. 构建 Docker 镜像
docker compose -f docker-compose.production.yml build

# 3. 启动所有服务（后台运行）
docker compose -f docker-compose.production.yml up -d

# 4. 等待服务启动（30-60秒）
sleep 30

# 5. 查看服务状态
docker compose -f docker-compose.production.yml ps

# 6. 查看日志（如果有问题）
docker compose -f docker-compose.production.yml logs -f
```

## 🌐 步骤 8: 配置 Nginx 反向代理（可选）

如果需要通过域名访问，配置 Nginx：

```bash
# 1. 安装 Nginx
sudo apt-get install -y nginx

# 2. 创建 Nginx 配置
sudo nano /etc/nginx/sites-available/redpacket
```

**配置内容示例：**

```nginx
server {
    listen 80;
    server_name 你的域名.com;

    # Web Admin
    location /admin {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # MiniApp API
    location /api {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # Frontend
    location / {
        proxy_pass http://127.0.0.1:3001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

```bash
# 3. 启用配置
sudo ln -s /etc/nginx/sites-available/redpacket /etc/nginx/sites-enabled/

# 4. 测试配置
sudo nginx -t

# 5. 重启 Nginx
sudo systemctl restart nginx
```

## ❓ 常见问题

### 问题 1: Docker 命令需要 sudo

**解决方案：**

```bash
# 重新登录 SSH 或执行
newgrp docker
```

### 问题 2: 端口被占用

**解决方案：**

```bash
# 检查端口占用
sudo netstat -tlnp | grep -E ':(8000|8080|3001|5432)'

# 停止占用端口的服务或修改 docker-compose.production.yml 中的端口映射
```

### 问题 3: 服务启动失败

**解决方案：**

```bash
# 查看详细日志
docker compose -f docker-compose.production.yml logs [服务名]

# 检查环境变量
cat .env.production

# 检查配置文件
docker compose -f docker-compose.production.yml config
```

### 问题 4: 数据库连接失败

**解决方案：**

```bash
# 检查数据库容器状态
docker compose -f docker-compose.production.yml ps db

# 检查数据库日志
docker compose -f docker-compose.production.yml logs db

# 检查环境变量中的密码
grep POSTGRES_PASSWORD .env.production
```
