# 🔧 解決 Git 克隆文件缺失問題

## 📊 當前狀態

從終端輸出看到：
- ✅ Git clone 成功（629 個對象）
- ✅ `.env.production` 已恢復
- ❌ `docker-compose.production.yml` 不存在
- ❌ `deploy/` 目錄不存在
- ❌ `frontend-next/` 目錄不存在

**這說明克隆不完整或文件被忽略了！**

---

## 🔍 診斷步驟

首先，在服務器上執行診斷命令，查看實際情況：

```bash
cd /opt/redpacket

# 1. 檢查當前目錄內容
ls -la

# 2. 檢查 Git 狀態
git status

# 3. 檢查分支
git branch -a

# 4. 檢查遠程倉庫
git remote -v

# 5. 檢查 .gitignore
cat .gitignore

# 6. 列出所有文件
find . -maxdepth 2 -type f -o -type d | head -30

# 7. 列出 Git 倉庫中的所有文件
git ls-files | head -50
```

---

## 🔧 解決方案

### 方案 1: 檢查分支並切換

可能克隆到了錯誤的分支：

```bash
cd /opt/redpacket

# 檢查所有分支
git branch -a

# 切換到 master 分支（如果不在）
git checkout master

# 拉取最新代碼
git fetch origin
git reset --hard origin/master

# 檢查文件
ls -la docker-compose.production.yml
ls -la deploy/
```

### 方案 2: 強制拉取所有文件

```bash
cd /opt/redpacket

# 強制重置到遠程 master
git fetch origin master
git reset --hard origin/master
git clean -fd

# 檢查文件
ls -la
```

### 方案 3: 檢查文件是否真的不存在

可能文件存在但被隱藏了：

```bash
cd /opt/redpacket

# 查找 docker-compose 文件
find . -name "docker-compose*.yml" -type f

# 查找 deploy 目錄
find . -name "deploy" -type d

# 列出所有文件（包括隱藏文件）
find . -type f | grep -v ".git" | head -50
```

### 方案 4: 直接從 GitHub 下載關鍵文件

如果 Git 倉庫有問題，可以直接下載：

```bash
cd /opt/redpacket

# 創建 deploy 目錄
mkdir -p deploy/scripts

# 下載 docker-compose.production.yml
curl -fsSL https://raw.githubusercontent.com/victor2025PH/hongbao20251025/master/docker-compose.production.yml -o docker-compose.production.yml

# 下載部署腳本
curl -fsSL https://raw.githubusercontent.com/victor2025PH/hongbao20251025/master/deploy/scripts/deploy_on_server.sh -o deploy/scripts/deploy_on_server.sh
chmod +x deploy/scripts/deploy_on_server.sh

# 檢查文件
ls -la docker-compose.production.yml
ls -la deploy/scripts/deploy_on_server.sh
```

## 💡 如果文件仍然缺失

如果執行上述命令後文件仍然缺失，可能是：

1. **Git 倉庫本身沒有這些文件**（可能在其他分支）
2. **文件被 .gitignore 忽略了**（不太可能）
3. **權限問題**（文件存在但不可見）

**解決方案：**

```bash
# 檢查所有分支
git branch -a

# 切換到所有分支查看
for branch in $(git branch -r | grep -v HEAD); do
    echo "檢查分支: $branch"
    git checkout $branch
    ls -la docker-compose.production.yml 2>/dev/null && echo "  ✅ 在 $branch 找到" || echo "  ❌ 在 $branch 未找到"
done
```
