# ✅ 執行完成後的檢查步驟

## 📊 修復腳本執行完成後

當修復腳本執行完成後，請按照以下步驟檢查和驗證：

---

## 🔍 步驟 1: 檢查服務狀態

```bash
cd /opt/redpacket

# 查看所有服務狀態
docker compose -f docker-compose.production.yml ps

# 預期輸出：所有服務應該顯示 "Up" 或 "healthy" 狀態
```

**如果服務未運行：**
```bash
# 查看詳細日誌
docker compose -f docker-compose.production.yml logs --tail 50

# 查看特定服務的日誌
docker compose -f docker-compose.production.yml logs web_admin
docker compose -f docker-compose.production.yml logs db
```

---

## 🧪 步驟 2: 測試健康檢查

```bash
# Web Admin 健康檢查
curl http://127.0.0.1:8000/healthz
# 預期輸出: {"ok": true, "ts": "..."}

# MiniApp API 健康檢查
curl http://127.0.0.1:8080/healthz
# 預期輸出: {"ok": true, "ts": "..."}

# 前端服務測試
curl http://127.0.0.1:3001 | head -20
# 預期輸出: HTML 內容
```

---

## 📋 步驟 3: 檢查環境變量

```bash
cd /opt/redpacket

# 檢查 .env.production 是否存在
ls -la .env.production

# 驗證環境變量
docker compose --env-file .env.production -f docker-compose.production.yml config | grep POSTGRES_USER
docker compose --env-file .env.production -f docker-compose.production.yml config | grep POSTGRES_DB
docker compose --env-file .env.production -f docker-compose.production.yml config | grep POSTGRES_PASSWORD

# 應該看到正確的值，而不是空白
```

---

## 🔍 步驟 4: 檢查 Docker 鏡像

```bash
# 查看所有鏡像
docker images | grep redpacket

# 應該看到：
# - redpacket/backend:latest
# - redpacket/frontend:latest
```

---

## 📊 步驟 5: 檢查服務日誌（無錯誤）

```bash
cd /opt/redpacket

# 查看所有服務的日誌（最後50行）
docker compose -f docker-compose.production.yml logs --tail 50

# 查看特定服務的日誌
docker compose -f docker-compose.production.yml logs db
docker compose -f docker-compose.production.yml logs web_admin
docker compose -f docker-compose.production.yml logs frontend
```

**查找錯誤：**
```bash
# 查找錯誤信息
docker compose -f docker-compose.production.yml logs 2>&1 | grep -i error
docker compose -f docker-compose.production.yml logs 2>&1 | grep -i fail
```

## 🚨 如果仍然有問題

### 問題 1: 服務未啟動

```bash
# 查看詳細日誌
docker compose -f docker-compose.production.yml logs

# 檢查配置文件
docker compose --env-file .env.production -f docker-compose.production.yml config

# 嘗試手動啟動單個服務
docker compose --env-file .env.production -f docker-compose.production.yml up -d db
docker compose --env-file .env.production -f docker-compose.production.yml up -d web_admin
```

### 問題 2: 健康檢查失敗

```bash
# 檢查服務是否正在運行
docker compose -f docker-compose.production.yml ps

# 檢查端口是否被占用
sudo netstat -tlnp | grep -E ':(8000|8080|3001|5432)'

# 檢查容器內部的服務
docker compose -f docker-compose.production.yml exec web_admin curl http://localhost:8000/healthz
```

### 問題 3: 環境變量仍然未設置

```bash
# 檢查 .env.production 文件格式
cat .env.production | grep POSTGRES

# 確保文件格式正確（沒有特殊字符）
file .env.production

# 手動設置環境變量測試
export $(cat .env.production | grep -v '^#' | xargs)
docker compose --env-file .env.production -f docker-compose.production.yml config
```
